# 基金交易账簿 — 移动端改版 Prompt

> 将此文档作为 Claude 的 prompt 输入，即可对任何基金/持仓类应用的移动端进行相同模式的改版。

---

## 改版目标

将原有的 2 页面（交易录入 + 持仓展示）响应式应用，重构为移动端优先的 4 页面架构，解决以下问题：
- 移动端表单拥挤、日期和净值输入框重叠
- 交易流水使用宽表需横向滑动，体验差
- 持仓页信息密度过高，关键指标不突出
- 新增调仓计算器功能

---

## 1. 页面架构变更

### Before（旧版）
```
2 页面，底部 2 Tab
  - 交易录入（表单 + 表格）
  - 持仓展示（汇总 + 表格）
```

### After（新版）
```
4 页面，底部 4 Tab
  - 录入 — 全屏表单，无表格
  - 流水 — 卡片列表，无横向滚动
  - 汇总 — 指标卡片网格
  - 明细 — 基金卡片列表 + 调仓计算器
```

### 底部 TabBar 设计
| 图标 | 标签 | 页面 |
|------|------|------|
| + | 录入 | 交易录入页 |
| 网格 | 流水 | 交易流水页 |
| 趋势线 | 汇总 | 持仓汇总页 |
| 圆+i | 明细 | 持仓明细页 |

### PC 端
保持顶部导航 + 原有的表格式数据密度布局，与移动端指标定义同步更新。

## 2. 页面一：交易录入页

### 布局规范
- 全屏表单，无左右分栏
- 顶部：页面标题
- 中部：表单区域
- 底部：固定操作栏（`position:sticky;bottom:0`），包含保存 + 重置按钮，防止键盘弹出时被遮挡
- 保存按钮占 75% 宽度，重置占 25%，`display:grid;grid-template-columns:3fr 1fr`

### 表单字段
| 字段 | 组件 | 交互 |
|------|------|------|
| 交易类型 | Segmented Control（4 态） | 一行，切换后动态改变下方字段 |
| 基金代码 | 文本输入 | 6 位数字，失焦后联想基金名称 |
| 基金名称 | 只读展示 | 联想失败时变为可编辑 |
| 交易日期 | 日期选择器 | 默认当天 |
| 单位净值 | 数字输入（可选） | 保留 4 位小数，已填时实时预览份额/金额 |
| 购买金额/卖出份额 | 数字输入 | 根据交易类型切换显示 |
| 代销渠道 | 下拉选择 | 支付宝/天天基金/银行/券商/其他 |
| 交易费率 | 数字输入（%） | 默认 0.15%，保留 3 位小数 |

### 关键约束
- **输入框高度 ≥ 44px，font-size ≥ 16px**（防止 iOS 自动缩放）
- 输入框采用固定网格宽度，禁止随内容动态拉伸
- 表单底部固定区域动态显示「预计确认份额：XXX / 预计到账金额：XXX」
- 数字输入框唤起数字键盘，日期唤起原生日期选择器

## 3. 页面二：交易流水页

### 卡片式列表设计（移动端，**禁止横向滚动**）

#### 卡片首屏（核心信息，必须突出）
| 信息 | 展示方式 | 说明 |
|------|---------|------|
| 基金名称 | 首行大字号（15px，600 weight） | 最优先信息 |
| 基金代码 | 名称下方小字灰色（11px） | 辅助信息 |
| 买卖方向 | 标签/徽章 | 买入=深蓝，卖出=橙色，分红=紫色，再投资=紫色 |
| 买卖金额/份额 | 大字号（18px，600 weight） | 卖出已确认时显示到账金额（元），否则显示份额 |
| 状态 | 状态标签 | 已确认=蓝色，待回填=橙色 |
| 日期 | 小字灰色 | 弱化到右下角 |

#### 金额颜色规则
- **按确认状态二分，不按交易类型**：
  - 已确认 → 深蓝（`--accent`）
  - 待回填 → 橙色（`--warn`）

#### 点击展开后（弱化信息）
- 交易日期、单位净值、确认份额、代销渠道、交易费率、净值来源
- 操作按钮：回填/编辑（待回填时显示回填，已确认时显示编辑）、删除

### 过滤
- 顶部过滤胶囊：全部 / 已确认 / 待回填（3 个，不包含已清仓）
- **已清仓是持仓维度概念，不出现在交易流水的状态中**

### 批量操作
- 底部「一键回填净值」按钮，仅当存在待回填记录时显示
- 空状态：「暂无交易记录，去录入第一笔」+ 跳转按钮

### PC 端
保留表格式布局（9 列），无横向滚动的卡片式仅在移动端使用。

### 卡片式列表设计（移动端，**禁止横向滚动**）

## 6. 调仓计算器（新增功能）

### 触发方式
- 持仓明细页每张基金卡片展开后，点击「调仓计算器」按钮
- 底部弹出面板（Bottom Sheet），圆角，有拖拽手柄

### 面板内容
| 字段 | 类型 | 说明 |
|------|------|------|
| 当前持仓市值 | 只读 | 自动带入 |
| 当前盈亏比例 | 只读 | 自动带入 |
| 当日预估净值 | 只读 | 使用 `gsz`（estimateNav） |
| 操作类型 | Segmented Control | 买入 / 卖出，默认买入 |
| 买入金额/卖出份额 | 数字输入 | 根据类型切换 |
| 交易费率 | 数字输入（%） | 默认 0.15% |

### 计算逻辑
```
买入：新增份额 = 买入金额×(1−费率) / 预估净值
      调仓后份额 = 当前份额 + 新增份额
      调仓后成本 = 当前成本 + 买入金额×(1−费率)

卖出：卖出成本 = (卖出份额/当前份额) × 当前成本
      调仓后份额 = 当前份额 − 卖出份额
      调仓后成本 = 当前成本 − 卖出成本

调仓后盈亏比例 = (调仓后份额×预估净值 − 调仓后成本 + 已实现盈亏 + 累计分红) / 调仓后成本
```

### 结果展示（6 行）
| 行 | 内容 |
|----|------|
| 调仓后份额 | 数字 |
| 调仓后成本 | 金额 |
| 调仓后成本单价 | newCost/newShares |
| 成本 vs 净值 | **(成本单价−预估净值)/预估净值** → 正值=浮亏（绿），负值=浮盈（红） |
| 调仓后盈亏比例 | 百分比，涨红跌绿 |
| 盈亏比例变化 | 调仓后−调仓前，涨红跌绿 |

### 重要约束
- 仅供模拟测算，**不产生真实交易记录**
- 底部提示：「* 仅供模拟测算，不产生真实交易记录」

## 8. 数据模型变更

### 交易状态简化
```
Before: confirmed | pending | cleared
After:  confirmed | pending
```
- `cleared` 从交易状态中移除
- 已清仓判断改为持仓层面：当某基金 `shares ≤ 0` 时，`computeHoldings()` 自动标记 `cleared:true`
- 已清仓的基金在持仓明细页灰度显示，折叠在底部

### 盈亏指标
```
删除：累计分红（汇总页不再单独展示）
新增：当日预估盈亏金额 = Σ(持仓份额 × (gsz − dwjz))
改名：总盈亏率 → 持仓盈亏比例
```

## 10. 关键交互流程

### 录入（未知净值）→ 待回填 → 一键回填 → 持仓更新
1. 用户在录入页填写交易，不填净值 → 保存后状态 = `pending`，橙色标签
2. 流水页显示橙色「待回填」标签，金额为橙色
3. 点击「一键回填净值」→ 扫描所有 pending 记录 → 按基金代码匹配预设净值数据 → 自动填入
4. Toast 提示成功/失败条数
5. 持仓汇总和明细页面联动更新

### 实时估值刷新
1. 用户点击「刷新全部估值」或单基金 ⟳ 按钮
2. 模拟更新 `gsz`（实时估值）和 `gszzl`（涨跌幅）
3. 汇总页当日预估盈亏 + 明细页涨跌列同步刷新
4. 按钮旋转动画 0.8s

- [ ] 无横向滚动（移动端卡片式，PC 端 `overflow-x:auto` 仅表格）

- [ ] 金额颜色基于确认状态，非交易类型
- [ ] 交易流水中不含「已清仓」状态

- [ ] 无 emoji 作为功能图标
- [ ] 无 lorem ipsum 或占位文案
