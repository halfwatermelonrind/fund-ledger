# 基金交易账簿 - 移动端页面重构设计需求

## 1. 重构背景

当前移动端为 2 页面结构（交易录入 + 持仓展示），因屏幕尺寸限制导致：
- 交易录入页表单拥挤，日期输入框与净值输入框重叠
- 交易流水使用宽表，需横向滑动，体验差
- 持仓页信息密度过高，关键指标不突出

**目标**：将移动端重构为 4 个独立页面，信息分层，每页聚焦单一任务，杜绝横向滑动。

---

## 2. 页面架构（移动端）

底部固定 TabBar，4 个入口：

| Tab 图标 | 标签 | 对应页面 | 核心任务 |
|---------|------|---------|---------|
| 录入 | 录入 | 交易录入页 | 快速录入一笔交易 |
| 流水 | 流水 | 交易流水页 | 查看、筛选、编辑历史交易 |
| 汇总 | 汇总 | 持仓汇总页 | 看整体账户盈亏概览 |
| 明细 | 明细 | 持仓明细页 | 看每只基金详情、排序、模拟调仓 |

**PC 端**：保持顶部导航，页面内容与移动端一致（指标定义同步更新）。

---

## 3. 页面一：交易录入页

### 3.1 布局
- 全屏表单，无左右分栏。
- 顶部：页面标题「交易录入」+ 快捷操作区（如「批量初始化」入口）。
- 中部：表单区域。
- 底部：固定操作栏（保存按钮 + 重置按钮），避免键盘弹出时按钮被遮挡。

### 3.2 表单字段与交互

| 字段 | 组件 | 说明 |
|------|------|------|
| 交易类型 | 四态 Segmented Control（买入/卖出/现金分红/红利再投资） | 占据一行，点击切换后动态改变下方字段 |
| 基金代码 | 文本输入 + 右侧联想浮层 | 输入 6 位代码，失焦后自动联想基金名称并回填到下一行 |
| 基金名称 | 只读展示（或允许手动修正） | 联想失败时变为可编辑输入框 |
| 交易日期 | 日期选择器 | **格式：YY/MM/DD**（如 26/06/20），缩短宽度 |
| 单位净值 | 数字输入 | 保留 4 位小数，**输入框宽度固定**（如占 40% 行宽），不随内容动态调整 |
| 购买金额/卖出份额 | 数字输入 | 根据交易类型切换标签，买入时显示「购买金额」，卖出时显示「卖出份额」 |
| 代销渠道 | 下拉选择 | 支付宝 / 天天基金 / 银行 / 券商 / 其他 |
| 交易费率 | 数字输入（%） | 默认 0.15%，保留 3 位小数 |

### 3.3 关键修复
- **日期输入框**：从 YYYY年MM月DD日 改为 YY/MM/DD 格式，缩短视觉宽度。
- **输入框宽度**：所有输入框采用**固定网格宽度**（如 50%/50% 或 60%/40% 分栏），**禁止随内容动态拉伸**。
- **实时预览**：表单底部固定区域，动态显示「预计确认份额：XXX」或「预计到账金额：XXX」。
- **键盘适配**：数字输入框唤起数字键盘，日期输入框唤起日期选择器（原生或自定义）。

## 4. 页面二：交易流水页

### 4.1 布局
- 顶部：标题「交易流水」+ 过滤胶囊（全部 / 已确认 / 待回填 / 已清仓）。
- 中部：纵向列表，**禁止横向滑动**。
- 底部：「一键回填净值」按钮（仅当有待回填记录时显示）。

### 4.2 列表项设计（卡片式或行式）

**每行/每张卡片展示的核心信息（必须突出）**：

| 信息 | 展示方式 | 说明 |
|------|---------|------|
| 基金名称 | 首行大字号 | 如「华夏成长混合」 |
| 基金代码 | 名称右侧小字灰色 | 如「000001」 |
| 买卖方向 | 标签/徽章 | 买入（蓝色）、卖出（橙色）、分红（紫色）、初始持仓（灰色） |
| 买卖金额/份额 | 大字号，金额带¥符号 | 如「¥10,000」或「5,000 份」 |
| 状态 | 状态标签 | 已确认（绿）/ 待回填（橙）/ 已清仓（灰） |

**弱化展示的信息（点击展开后可见）**：
- 交易日期
- 单位净值
- 确认份额
- 代销渠道
- 交易费率
- 操作按钮（编辑 / 删除）

### 4.3 交互
- 点击行：展开/折叠详情，展示弱化信息和操作按钮。
- 左滑（可选）：快捷删除（二次确认）。
- 排序：默认按交易日期倒序，最新在上。
- 空状态：「暂无交易记录，去录入第一笔」+ 跳转按钮。

## 5. 页面三：持仓汇总页

### 5.1 布局
- 顶部：标题「持仓汇总」。
- 中部：指标卡片网格（2 列或 1 列大卡片）。
- 底部：「刷新估值」按钮（刷新所有持仓基金的最新净值和实时估值）。

### 5.2 指标卡片（同步更新 PC 版）

| 指标 | 展示 | 说明 |
|------|------|------|
| 总市值 | 大字号金额 | 所有持仓基金市值加总 |
| 总成本 | 中字号金额 | 所有持仓成本加总 |
| 总浮动盈亏 | 金额 + 颜色 | 涨红跌绿 |
| 总已实现盈亏 | 金额 + 颜色 | 涨红跌绿 |
| ~~累计分红~~ | ~~已删除~~ | ~~不再展示~~ |
| 总盈亏 | 金额 + 颜色 | 涨红跌绿 |
| **持仓盈亏比例** | 百分比 + 颜色 | 原「总盈亏率」改名，涨红跌绿 |
| **当日预估盈亏金额** | 新增，金额 + 颜色 | 所有持仓基金当日预估盈亏加总 = 持仓份额 * (estimateNav - latestNav)，涨红跌绿 |

### 5.3 卡片设计
- 每个指标一个卡片，白色背景，圆角，轻微阴影。
- 金额使用等宽字体，正负用颜色区分。
- **当日预估盈亏金额**卡片可突出显示（如顶部通栏或更大字号），因为这是用户最关心的"今天赚了/亏了多少钱"。
- 预估盈亏为 0 或无法获取时（如 QDII），显示「--」灰色。

## 6. 页面四：持仓明细页

### 6.1 布局
- 顶部：标题「持仓明细」+ 排序选择器（下拉或按钮组）。
- 中部：纵向卡片列表，**禁止横向滑动**。
- 底部：无固定按钮（调仓计算器在卡片内触发）。

### 6.2 排序选择器
支持按以下维度排序，点击切换升序/降序：

| 排序维度 | 说明 |
|---------|------|
| 持仓市值 | 默认降序，市值大的在前 |
| 持仓盈亏比例 | 按总盈亏率排序 |
| 当日预估净值涨跌幅 | 按 estimateChange 排序 |

### 6.3 每只基金的卡片设计

**卡片首屏（核心信息，必须突出）**：

| 信息 | 展示方式 | 说明 |
|------|---------|------|
| 基金名称 | 卡片顶部大字号 | 如「华夏成长混合」 |
| 基金代码 | 名称下方小字灰色 | 如「000001」 |
| 持仓市值 | 大字号金额 | 如「¥14,070」 |
| 持仓盈亏比例 | 百分比，大字号 | 涨红跌绿 |
| 当日预估净值涨跌幅 | 百分比，带颜色 | 涨红跌绿，QDII 无数据时显示「--」 |

**点击展开后（弱化信息）**：
- 持仓份额
- 持仓成本
- 成本单价
- 最新净值（已确认）
- 实时估值（estimateNav）
- 浮动盈亏
- 已实现盈亏
- 累计分红（弱化展示，不删除数据）
- 操作按钮：刷新估值 / 更新净值 / 调仓计算器

### 6.4 调仓计算器（新增功能）

**触发方式**：每张卡片展开后，底部显示「调仓计算器」按钮，点击后底部弹出面板。

**面板内容**：

| 字段 | 组件 | 说明 |
|------|------|------|
| 当前持仓市值 | 只读 | 自动带入 |
| 当前持仓盈亏比例 | 只读 | 自动带入 |
| 当日预估净值 | 只读 | 自动带入（estimateNav） |
| 操作类型 | 切换按钮（买入 / 卖出） | 默认买入 |
| 买入金额 | 数字输入 | 买入时必填 |
| 卖出份额 | 数字输入 | 卖出时必填 |
| 交易费率 | 数字输入（%） | 默认 0.15% |

**计算结果（实时预览）**：
- 调仓后持仓份额
- 调仓后持仓成本
- 调仓后持仓盈亏比例
- 与当前盈亏比例的差值（如「+2.3%」或「-1.5%」）

**计算逻辑**：
```
买入时：
  新增份额 = 买入金额 * (1 - feeRate) / estimateNav
  调仓后总份额 = 当前份额 + 新增份额
  调仓后总成本 = 当前成本 + 买入金额 * (1 - feeRate)
  调仓后盈亏比例 = (调仓后总份额 * estimateNav - 调仓后总成本) / 调仓后总成本

卖出时：
  卖出金额 = 卖出份额 * estimateNav * (1 - feeRate)
  调仓后总份额 = 当前份额 - 卖出份额
  调仓后总成本 = 当前成本 - (卖出份额 / 当前份额) * 当前成本
  调仓后盈亏比例 = (调仓后总份额 * estimateNav - 调仓后总成本) / 调仓后总成本
```

**注意**：调仓计算器**不产生真实交易记录**，仅用于模拟计算，帮助用户决策。用户确认后仍需回到「交易录入页」正式录入。

## 7. 设计规范（沿用品牌令牌）

- 颜色：涨红 #dc2626，跌绿 #16a34a，平灰 #9ca3af，主色 #1e3a8a，警告橙 #f59e0b
- 字体：系统默认 + 等宽数字
- 圆角：4/8/12px
- 间距：移动端适当加大行高和卡片间距，避免拥挤
- 触控：按钮最小 44px，输入框 font-size >= 16px 防 iOS 缩放

---

## 8. 交付物

1. 4 个页面的移动端高保真原型（含展开/折叠状态）。
2. 调仓计算器面板原型（底部弹出）。
3. 底部 TabBar 设计（4 个入口，含激活态）。
4. 关键交互流程：录入 → 流水 → 汇总 → 明细 → 调仓计算器。
5. PC 端同步更新：持仓汇总页指标卡片、持仓明细页排序逻辑。
