# 基金交易账簿 — 样式与交互打磨 Prompt

> 将此文档发给 Claude，对 `h5.html`（移动端）和 `pc.html`（桌面端）进行样式统一和交互打磨。

---

## 背景

已完成的改版：原有的 2 页面响应式应用已重构为移动端 4 Tab 架构（录入/流水/汇总/明细），PC 端保留 2 Tab 表格式布局。数据逻辑和交互流程已就位，现在需要打磨视觉一致性和交互细节。

---

## P0 — 必须修复

### 1. 触控目标尺寸不达标

iOS HIG 要求最小触控目标 44×44pt。以下组件低于这个标准：

| 组件 | 当前尺寸 | 要求 | 文件 |
|------|---------|------|------|
| `.seg-btn` Segmented Control | `min-height:40px` | `min-height:44px` | h5.html |
| `.btn-sm` 小按钮 | `height:36px` | 至少 `height:40px`，如果空间允许到 44px | h5.html |
| `.btn-xs` 超小按钮 | `height:30px` | 至少 `height:36px`，仅用于表格行内操作 | h5.html |
| `.fpill` 过滤胶囊 | `min-height:36px` | `min-height:40px` | h5.html |
| `.sort-btn` 排序按钮 | `min-height:36px` | `min-height:40px` | h5.html |
| `.rf-btn` 刷新按钮 | `30×30px` | 至少 `36×36px` | h5.html |

### 2. 硬编码颜色值未使用设计令牌

以下颜色直接写在 CSS/JS 中，应改为 `:root` 令牌或语义类名：

| 位置 | 当前值 | 问题 | 建议 |
|------|--------|------|------|
| 红利再投资类型标签 | `#7e22ce` | 硬编码紫色 | 新增 `--type-reinvest: #7e22ce` 令牌 |
| 红利再投资背景 | `#f3e8ff` | 硬编码浅紫 | 新增 `--type-reinvest-bg: #f3e8ff` 令牌 |
| 待回填标签文字 | `#92400e` | 硬编码深橙 | 新增 `--warn-text: #92400e` 令牌 |
| 按钮 hover 态 | `#1e40af` | 深蓝 hover | 可用 `color-mix(in oklch, var(--accent), #000 15%)` |
| 按钮 hover 态 | `#d97706` | 橙 hover | 可用 `color-mix(in oklch, var(--warn), #000 15%)` |
| 表格行 hover | `#f8fafc` | 浅灰蓝 | 新增 `--row-hover: #f8fafc` 令牌或使用 `color-mix` |

### 3. h5.html 和 pc.html 的 CSS 类名不统一

同一个项目的两个文件使用了两套完全不同的 CSS 类命名：

| 组件 | h5.html | pc.html |
|------|---------|---------|
| 主按钮 | `.btn-primary` | `.btn-p` |
| 次按钮 | `.btn-secondary` | `.btn-s` |
| 幽灵按钮 | `.btn-ghost` | `.btn-g` |
| 危险按钮 | `.btn-danger` | `.btn-d` |
| 警告按钮 | `.btn-warn` | `.btn-w` |
| 已确认徽章 | `.badge-confirmed` | `.bc` |
| 待回填徽章 | `.badge-pending` | `.bp` |

**要求**：统一为 h5.html 的描述性命名风格，或至少保持两个文件可互换。

## P1 — 应该修复

### 4. 移动端底部 TabBar 图标缺乏视觉份量

当前图标使用 1.8px 描边 SVG，在 22×22px 尺寸下显得单薄。对比真实金融类 App（支付宝、天天基金），Tab 图标通常用填充式或更粗描边。

**要求**：
- 将 `stroke-width` 从 `1.8` 改为 `2.2`
- 激活态 Tab 图标改为填充色（`fill:var(--accent)`），非激活态保持空心
- 或者改用更贴近金融场景的图标语意（录入=/笔+纸，流水=/列表，汇总=/饼图，明细=/放大镜）

### 5. 卡片阴影过于微弱

当前卡片阴影 `0 1px 2px rgba(0,0,0,.05)` 在白色背景上几乎不可见，卡片与背景的层次感不足。

**要求**：
- 移动端卡片：`0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)`
- PC 端使用现有的 `0 4px 12px rgba(0,0,0,.08)` 即可

### 6. 交易流水卡片缺少「左滑删除」手势

规格中提到"左滑（可选）：快捷删除（二次确认）"。当前仅有点击展开后在详情中删除。

**要求**：
- 移动端交易流水卡片（`.tx-card`）添加 `touch-action:pan-y` + 左滑露出红色删除按钮
- 滑动距离超过 60px 时触发删除确认（Toast 二次确认或弹出 ActionSheet）
- 如果实现复杂度高，至少加一个明确的删除图标而非隐藏在展开详情中

### 7. 汇总页面的「当日预估盈亏」缺少无数据态

非交易时段当前显示"非交易时段"文字标签，但卡片内部仍显示 `±0.00 元`（渲染为 pnl-zero 灰色）。当所有基金都没有 gsz 数据时，应显示 `--`。

**要求**：
- 非交易时段或数据不可用时，「当日预估盈亏」卡片数值显示为 `--`（灰色）
- 卡片底部小字改为"非交易时段，暂无估值数据"

## P2 — 建议优化

### 8. 输入框聚焦态反馈太弱

当前聚焦态 `box-shadow:0 0 0 3px rgba(30,58,138,.1)` 在浅色背景上不够明显。

**要求**：
- 聚焦环改为 `0 0 0 2px var(--accent), 0 0 0 4px rgba(30,58,138,.15)`
- 或使用更明显的 `border-color:var(--accent)` + `outline:2px solid var(--accent-light)`

### 9. Toast 位置不够醒目

当前 Toast 在右上角，在单手操作时（右手持机）可能被拇指遮挡。

**要求**：
- 移动到顶部居中位置（`top:16px;left:50%;transform:translateX(-50%)`）
- 或者移到屏幕中间偏上（`top:20%;left:50%`），更接近系统级通知的体感

### 10. 调仓计算器 Bottom Sheet 的关闭体验

当前只能通过点击遮罩层或底部「关闭」按钮关闭。缺少下滑手势关闭。

**要求**：
- 添加 `touchstart/touchmove/touchend` 监听，下滑距离超过 80px 时自动关闭
- 或者添加一个更明显的顶部拖拽手柄指示器（当前手柄 36×4px 偏小）

### 11. 汇总页指标卡片节奏

当前 7 张卡片中「当日预估盈亏」是通栏（`grid-column:1/-1`），其余 6 张是 2 列网格。在 2 列布局下，第 7 张「持仓盈亏比例」会单独占据一行底部，视觉不平衡。

**要求**：
- 移动端（2 列）：将「总盈亏」和「持仓盈亏比例」合并为一张卡片（如：上方文字"总盈亏"，下方大字号比例）
- 或者将指标精简为 6 个，保持 2 列对称

### 12. 数字字体回退链

当前 `--mono` 字体链中 `SF Mono` 在非 Apple 设备上会回退到 `JetBrains Mono`，但该字体可能未安装。

**要求**：
- 在 `--mono` 末尾追加更通用的回退：`'Courier New', monospace`
- 或使用 `ui-monospace` 作为首选（已在链中），移除对特定字体的依赖

### 13. PC 端表格行 hover 色不一致

h5.html 的 PC 表格 hover 使用 `#f8fafc`，pc.html 使用 `#f8fafc`。两者一致，但都未定义为 token。应统一使用 `--row-hover` 令牌。

---

## 不需要改的（保持现状）

- 品牌令牌值（`--accent:#1e3a8a`, `--gain:#dc2626`, `--loss:#16a34a` 等）— 与品牌规范一致
- A 股涨红跌绿配色惯例 — 所有盈亏显示均正确
- 调仓计算器公式 — 已验证与规格一致
- 数据模型和状态管理逻辑 — 已确认正确
- 页面结构和导航架构 — 符合改版目标

---

## 验证清单

改完后逐项确认：

- [ ] 所有触控目标 ≥ 44px（或 ≥ 36px 对于行内小操作）
- [ ] 无裸 hex 颜色值在非 `:root` 位置（hover 态、badge 文字色除外，但这些应 token 化）
- [ ] h5.html 和 pc.html 使用相同的 CSS 类命名体系
- [ ] 移动端卡片在浅色模式下有明显的层次感
- [ ] 非交易时段下的所有估值相关数据显示 `--` 而非假数据
- [ ] Tab 图标在激活/非激活态有清晰的视觉差异
- [ ] Toast 位置方便单手操作时看到
